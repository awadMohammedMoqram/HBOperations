## 1. مبادئ التصميم العامة

### 1.1 المبادئ الأساسية

| المبدأ | الوصف | التطبيق |
|--------|-------|---------|
| **Need-to-Know** | يرى المستخدم فقط البيانات اللازمة لأداء وظيفته | فلترة على مستوى الـ Query وليس الـ View |
| **Least Privilege** | الحد الأدنى من الصلاحيات بشكل افتراضي | لا تقارير افتراضية للموظف العادي إلا تقاريره الذاتية |
| **Data Minimization** | إخفاء الحقول الحساسة في تقارير الإحصاءات | استبدال الأسماء بأرقام مرجعية للأدوار الدنيا |
| **Defense-in-Depth** | تطبيق الفلترة على ثلاث طبقات | UI + Endpoint + Database Query |
| **Audit Everything** | تسجيل كل عملية تصدير في سجل التدقيق | AuditService.LogAsync لكل export |
| **Watermarking** | علامة مائية بهوية المستخدم على كل PDF | منع الالتقاط/التسريب غير المُرخّص |

### 1.2 تصنيف البيانات

```
🟢 عام (Public):           أسماء الفروع، الإدارات، الإحصاءات الإجمالية
🟡 داخلي (Internal):        أسماء الموظفين، أوقات المعالجة، أنواع المعاملات
🟠 سري (Confidential):     محتوى المعاملات، الملاحظات، الموضوعات
🔴 سري للغاية (Restricted): ملاحظات الرفض، تعديلات Admin، ملاحظات الامتثال
```

### 1.3 معادلة الوصول

```
User_Visible_Data = (Role_Global_Access × All_Data) 
                  ∪ (Role_Local_Scope × Scoped_Data) 
                  ∪ (User_Personal_Involvement × Personal_Data)
                  ─ Restricted_Fields_For_Role
```

## 2. مصفوفة الصلاحيات الإجمالية

### 2.1 الأدوار العامة (Global Access — 8 أشخاص)

| الدور | الوصول | يرى | لا يرى |
|-------|--------|-----|---------|
| **SuperAdmin** | كامل + تعديل | كل شيء + سجل التدقيق + التعديلات الإدارية | — |
| **CEO** | كامل قراءة | كل التقارير + الإحصاءات الاستراتيجية | سجل التدقيق التفصيلي |
| **ITAdmin** | كامل + تقني | كل المعاملات + سجلات النظام + الأرشيف | — |
| **AssistantCEO** | كامل قراءة | كل التقارير الإدارية | سجل التدقيق |
| **Auditor** | كامل تدقيق | كل المعاملات + سجل التدقيق + الرفض + التعديلات | — |
| **ComplianceOfficer** | كامل امتثال | كل المعاملات + الرفض + الملاحظات الحساسة | — |
| **ShariahAuditor** | كامل شرعي | المعاملات المالية + التحويلات النقدية فقط | المستندات الإدارية البحتة |

### 2.2 الأدوار المحلية (Scoped Access)

| الدور | النطاق | الوصول | تقاريره |
|-------|--------|--------|---------|
| **DepartmentManager** | إدارة واحدة | معاملات إدارته + الموظفين | تقرير الإدارة + أداء الموظفين |
| **BranchManager** | فرع واحد | معاملات فرعه + الموظفين | تقرير الفرع + أداء الموظفين |
| **OfficeManager** | مكتب واحد | معاملات مكتبه | تقرير المكتب |
| **DepartmentStaff** | شخصي + إدارة محدودة | معاملاته الذاتية فقط | تقرير ذاتي |
| **BranchStaff** | شخصي + فرع محدود | معاملاته الذاتية فقط | تقرير ذاتي |
| **AdminAffairsStaff** ⭐ | خاص | كل المعاملات `Sent`/`InTransit` | تقرير الشؤون الإدارية |

⭐ **ملاحظة:** AdminAffairsStaff = `DepartmentStaff` أو `DepartmentManager` ينتمي للإدارة بالكود `DEP-ADM`

## 3. مواصفات التقارير حسب الشخصية

### 🔴 3.1 SuperAdmin / CEO / AssistantCEO (الإدارة العليا)

#### نوع التقارير المتاحة:
| التقرير | الوصف | الإيقونة |
|---------|-------|---------|
| **التقرير التنفيذي الشامل** | لوحة إستراتيجية شاملة | 📊 |
| **تقرير الفروع** | كل الفروع مع المقارنات | 🏢 |
| **تقرير الإدارات** | كل الإدارات مع الأداء | 🏛️ |
| **تقرير الأداء الشامل** | جميع الموظفين والوحدات | ⚡ |
| **تقرير الاتجاهات الزمنية** | تحليل ربع/سنوي | 📈 |
| **تقرير المعاملات الحرجة** | المتأخرة + المرفوضة | ⚠️ |

#### المكونات في كل تقرير:
```
┌─────────────────────────────────────────────────┐
│ Header: شعار البنك + النقش + اسم التقرير        │
├─────────────────────────────────────────────────┤
│ Executive Summary (4 KPIs)                      │
│  • إجمالي المعاملات   • نسبة الإنجاز            │
│  • متوسط الزمن       • المعاملات الحرجة         │
├─────────────────────────────────────────────────┤
│ Charts:                                         │
│  • Trend Chart (آخر 12 شهر)                    │
│  • Comparison Bar Chart (الفروع/الإدارات)      │
│  • Status Distribution Pie                      │
├─────────────────────────────────────────────────┤
│ Detailed Tables:                                │
│  • Top/Bottom 5 Performers                      │
│  • Branch Ranking                               │
│  • Department Ranking                           │
├─────────────────────────────────────────────────┤
│ Strategic Insights:                             │
│  • تحليل الاتجاهات                              │
│  • التوصيات التشغيلية                           │
│  • الأنماط الموسمية                             │
├─────────────────────────────────────────────────┤
│ Footer: علامة مائية بهوية المُصدِر               │
└─────────────────────────────────────────────────┘
```

#### الحقول المعروضة:
- ✅ كل الإحصاءات والمجموعات
- ✅ أسماء الموظفين والوحدات
- ✅ أوقات المعالجة التفصيلية
- ✅ نسب الإنجاز والرفض
- ❌ محتوى المعاملات الفردية (في التقارير الإجمالية)

#### تنسيق PDF:
- **الحجم:** A4 Landscape للوحة التنفيذية، A4 Portrait للتقارير الفرعية
- **الألوان:** ألوان البنك الكاملة (Primary + Gold)
- **الرسوم البيانية:** مدمجة كصور SVG
- **عدد الصفحات:** 5-15 صفحة

#### تنسيق Excel:
- **عدد الأوراق:** 6-8 (Summary, Branches, Departments, Performance, Trends, Charts Data, Raw Data)
- **التنسيق الشرطي:** ملون حسب الأداء
- **Pivot Tables:** جاهزة مسبقاً

### 🔵 3.2 Auditor / ComplianceOfficer / ShariahAuditor (التدقيق والرقابة)

#### نوع التقارير المتاحة:
| التقرير | الوصف | الخاص بـ |
|---------|-------|----------|
| **سجل التدقيق الكامل** | كل تغييرات الحالة | Auditor + Compliance |
| **تقرير المعاملات المرفوضة** | مع أسباب الرفض الكاملة | Compliance |
| **تقرير التعديلات الإدارية** | تعديلات SuperAdmin | Auditor |
| **تقرير الانحرافات الزمنية** | المتأخرة عن SLA | Compliance |
| **تقرير المعاملات المالية** | تحويلات نقدية فقط | ShariahAuditor |
| **تقرير الامتثال للسياسات** | فحوصات الامتثال | Compliance |

#### المكونات الخاصة:

**أ) سجل التدقيق:**
```
┌────────────────────────────────────────────┐
│ تقرير سجل التدقيق - الفترة [من] إلى [إلى]  │
├────────────────────────────────────────────┤
│ ملخص:                                      │
│  • عدد الإجراءات: XXX                      │
│  • عدد المستخدمين: XX                      │
│  • نوع الإجراءات: [Create, Update, ...]    │
├────────────────────────────────────────────┤
│ جدول تفصيلي:                               │
│  | الوقت | المستخدم | الإجراء | IP | تفاصيل│
├────────────────────────────────────────────┤
│ تنبيهات أمنية:                             │
│  • محاولات وصول مشبوهة                     │
│  • تعديلات خارج ساعات العمل                │
│  • تكرار الإجراءات                         │
└────────────────────────────────────────────┘
```

**ب) تقرير المعاملات المرفوضة (Compliance):**
- يعرض **RejectionNote الكامل** (مخفي للأدوار الأخرى)
- تصنيف أسباب الرفض
- تحليل الاتجاهات (هل هناك إدارة معينة ترفض كثيراً؟)
- توصيات تحسين العمليات

**ج) تقرير الرقابة الشرعية:**
- فقط `TransactionType = CashTransfer`
- إخفاء البيانات غير المالية
- تحليل التوافق مع المعايير الشرعية
- تقرير الحالات المشكوك فيها

#### الحقول المعروضة:
- ✅ كل الحقول الحساسة (RejectionNote, AdminNote, AdminEditedBy)
- ✅ سجل التدقيق الكامل (Old/New JSON values, IP, UserAgent)
- ✅ Timestamps دقيقة
- ⚠️ بيانات شخصية للموظفين (مع إشعار قانوني في التقرير)

#### تنسيق خاص:
- **علامة "تقرير سري — للتدقيق فقط"** على كل صفحة
- **تشفير PDF** بكلمة مرور (اختياري)
- **رقم تسلسلي للتقرير** للتتبع
- **Watermark** بالاسم + التاريخ + ساعة التصدير

### 🟢 3.3 DepartmentManager (مدير إدارة)

#### نوع التقارير المتاحة:
| التقرير | الوصف |
|---------|-------|
| **تقرير الإدارة الشهري** | كل المعاملات الواردة/الصادرة لإدارته |
| **أداء موظفي الإدارة** | KPI لكل موظف تابع له |
| **تقرير المعاملات المعلّقة** | بحاجة متابعة |
| **تقرير الوقت** | متوسط زمن المعالجة في إدارته |
| **تقرير الحالات الحرجة** | المتأخرة + المرفوضة في إدارته |

#### المكونات:
```
┌────────────────────────────────────────┐
│ Header: اسم الإدارة + الفترة            │
├────────────────────────────────────────┤
│ KPIs (4 cards):                        │
│  • معاملات الإدارة  • نسبة الإنجاز      │
│  • متوسط الزمن    • المعلّقة           │
├────────────────────────────────────────┤
│ تحليل الموظفين (جدول):                  │
│  | الموظف | المعالجة | المكتملة | الزمن │
├────────────────────────────────────────┤
│ المعاملات الحرجة (آخر 10):              │
│  | المرجع | الموضوع⚠️ | المرسل | الحالة │
├────────────────────────────────────────┤
│ Footer: المُصدِر + الإدارة                │
└────────────────────────────────────────┘
```

⚠️ **الموضوع** يُعرض بشكل مختصر (أول 50 حرف) أو يُحجب إذا اعتُبر حساساً.

#### الحقول المعروضة:
- ✅ معاملات إدارته فقط (`SenderDepartmentId == myDeptId OR ReceiverDepartmentId == myDeptId`)
- ✅ أسماء موظفي إدارته
- ✅ Subject (مختصر)
- ⚠️ Description (فقط في تفاصيل المعاملة، ليس في التقرير الإجمالي)
- ❌ RejectionNote الكامل (يظهر فقط "مرفوضة" بدون السبب)
- ❌ AdminNote
- ❌ معاملات إدارات أخرى

#### الفلاتر المتاحة:
- الفترة الزمنية (أسبوع/شهر/ربع/سنة)
- نوع المعاملة
- الحالة
- الموظف (من إدارته فقط)
- ❌ الفرع (مخفي - ليس له صلاحية)

## 6. قواعد التصدير (PDF / Excel)

### 6.1 PDF

| القاعدة | التطبيق |
|---------|---------|
| **علامة مائية** | اسم المُصدِر + الوقت + IP — لكل صفحة |
| **رقم تسلسلي** | `RPT-YYYYMMDD-HHmmss-{Hash6}` لكل تقرير |
| **حماية بكلمة مرور** | اختياري للأدوار العليا، إجباري لتقارير التدقيق |
| **No-Print Marker** | للتقارير شديدة السرية (Compliance) |
| **حد أقصى للحجم** | 50 صفحة لتجنب التسريبات الكبيرة |
| **سجل التصدير** | كل تصدير يُسجَّل في AuditLog مع التفاصيل |

### 6.2 Excel

| القاعدة | التطبيق |
|---------|---------|
| **حماية الأوراق** | Sheet protection بكلمة مرور (read-only) |
| **حذف Macros** | لا VBA - فقط بيانات وصيغ |
| **علامة في الميتاداتا** | Custom Properties: `ExportedBy`, `ExportTime`, `ReportType` |
| **حد البيانات الخام** | Raw Data sheet متاحة فقط للأدوار العليا |
| **التشفير** | Excel password للتقارير الحساسة |

### 6.3 قواعد عامة

```csharp
// Pseudo-code للتحقق قبل التصدير
public async Task<byte[]> ExportReport(Guid userId, ReportType type, ExportFormat format)
{
    var user = await GetUserWithRoles(userId);
    var policy = new ReportAccessPolicy(user);

    // 1. التحقق من الصلاحية
    if (!policy.CanAccessReport(type))
        throw new ForbiddenException("Access denied to this report type");

    // 2. التحقق من حد التصدير اليومي (Rate Limiting)
    var todayExports = await CountUserExportsToday(userId);
    if (todayExports >= GetExportLimit(user))
        throw new RateLimitException("Daily export limit reached");

    // 3. بناء البيانات حسب النطاق
    var scope = policy.GetUserScope();
    var data = await ReportDataBuilder.Build(type, scope);

    // 4. تطهير البيانات الحساسة
    var sanitized = ReportSanitizer.Sanitize(data, user.Roles);

    // 5. توليد الملف
    var bytes = format switch
    {
        ExportFormat.Pdf => _pdfService.Generate(sanitized, user, watermark: true),
        ExportFormat.Excel => _excelService.Generate(sanitized, user, protected: true),
        _ => throw new NotSupportedException()
    };

    // 6. تسجيل في سجل التدقيق
    await _auditService.LogAsync("ReportExported", userId, new {
        ReportType = type, Format = format, RowCount = sanitized.Count,
        Scope = scope.ToString(), FileHash = ComputeHash(bytes)
    });

    return bytes;
}
```

## 7. التحكم في الوصول والتدقيق

### 7.1 طبقات الحماية

```
┌──────────────────────────────────────────────────┐
│ Layer 1: UI Authorization                        │
│  • [@attribute Authorize(Roles="...")]           │
│  • زر التصدير مرئي فقط للمصرح لهم                │
├──────────────────────────────────────────────────┤
│ Layer 2: Endpoint Authorization                  │
│  • RequireAuthorization() في Program.cs          │
│  • Role-based middleware                         │
├──────────────────────────────────────────────────┤
│ Layer 3: Query Filtering                         │
│  • IQueryable<Transaction> مع .Where(scope)      │
│  • Global Query Filters في DbContext             │
├──────────────────────────────────────────────────┤
│ Layer 4: Field Sanitization                      │
│  • ReportSanitizer قبل التوليد                   │
│  • DTO mapping يحذف الحقول غير المسموحة          │
├──────────────────────────────────────────────────┤
│ Layer 5: Output Watermarking                     │
│  • PDF watermark + metadata                      │
│  • Excel custom properties                       │
├──────────────────────────────────────────────────┤
│ Layer 6: Audit Logging                           │
│  • AuditService.LogAsync لكل تصدير               │
│  • SIEM-ready logs (JSON structured)             │
└──────────────────────────────────────────────────┘
```
