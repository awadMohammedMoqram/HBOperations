# اقتراح تصميم صفحة التقارير لكل شخصية

> هذا المستند يقترح أفضل تخطيط (Layout) لصفحة التقارير لكل دور في النظام، بحيث يرى كل مستخدم ما يناسب مسؤولياته مباشرةً دون تشتت.

---

## الفلسفة العامة

| المبدأ | التطبيق |
|--------|---------|
| **المعلومة الأهم أولاً** | كل دور يرى KPIs الأكثر أهمية لعمله في الأعلى |
| **قابلية الإجراء** | زرّ "عرض التفاصيل" أو link مباشر من كل جدول إلى صفحة المعاملة |
| **تصميم واحد — محتوى ديناميكي** | نفس الهيكل العام (Header → Cards → Charts → Table) لكنّ البيانات تتغير |
| **لا إفراط** | لا نعرض أقساماً فارغة — كل شيء مشروط بوجود بيانات |

## التوصيات التقنية للتنفيذ

### 1. مكوّن واحد + أقسام شرطية
```razor
@if (_scope.IsExecutive) { <ExecutiveSections /> }
@if (_scope.IsAuditor || _scope.IsCompliance) { <AuditTrailTable /> }
@if (_scope.IsBranchManager) { <StaffPerformanceTable /> }
<!-- ... إلخ -->
```

### 2. بديل أفضل: RenderFragment ديناميكي
بدلاً من if/else كثيرة، يمكن بناء `List<ReportSection>` ديناميكياً ثم عرضها:
```csharp
var sections = BuildSections(scope, selectedReportType);
// كل section = title + component/RenderFragment
```

### 3. جدول المعاملات كمكوّن مشترك
```razor
<TransactionTable Rows="@_transactions" 
                  ShowSearch="true"
                  ShowPagination="true"
                  PageSize="15"
                  Columns="@_visibleColumns" />
```
- `Columns` تتغير حسب الدور (المدقق يرى أعمدة أكثر، الموظف أقل)

### 4. CSS: لا تغيير — نفس التصميم الحالي
- الـ Cards + Charts + Tables موجودة ومصممة
- فقط نضيف/نخفي أقسام حسب الدور

### 5. أولوية التنفيذ
1. ✅ إضافة جدول المعاملات للأدوار التي لا تملكه حالياً (Executive + Default)
2. إضافة "أداء الموظفين" لمدير الفرع/الإدارة
3. إضافة "التنبيهات" للمدقق
4. تحسين ترتيب الأقسام حسب الدور (الأهم أولاً)
